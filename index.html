Croofx — The AI Execution Control Layer

Founder · Solo · Building in public
Deterministic enforcement for AI-generated code.
Every team using AI to write code has the same invisible problem: the AI generates patches probabilistically. But your CI pipeline, your production systems, your deployment contracts — they demand determinism. Right now developers are manually acting as that enforcement layer. Croofx automates it.
AI generates. Croofx enforces.
AI says: "I think this patch is correct."
Croofx says: "Validation failed. Merge blocked."

croofx validate --ci → FAIL → exit(1) → pipeline fails → merge blocked.
No configuration needed. No GitHub OAuth required. Works offline except for the AI call.
CLI · Python SHA256 determinism CI/CD enforcement Patch validator Model-agnostic v0.0.1 shipped
Core Enforcement Engine
Deterministic repo scanner with stable file ordering, SHA256-based hashing, CRLF normalization. Byte-for-byte identical output for unchanged repos. No timestamps, no randomness, no AI in the validator.
v0.0.1 → v1.0.0 in progress
Structured AI Contract Layer
LLM output normalized into strict JSON schemas before it ever reaches the validator. No loose parsing, no regex hacks. The AI's patch is a contract — not a suggestion.
v0.0.2 · Schema enforcement
Patch Simulation & Validator
Applies patches to in-memory copies, validates syntax, resolves imports, checks structural boundaries. Pure deterministic logic — zero AI inside the validator. Returns PASS → exit(0) or FAIL → exit(1).
v0.9.0 pre-release · THE NUCLEUS
Roadmap: SaaS + Governance
v2.0.0 brings secure sandbox execution, team-level AI policy enforcement, dashboard, audit logs. Croofx becomes the governance layer for AI usage across engineering organizations.
